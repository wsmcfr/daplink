# 无线 DAP 烧录器中配版本方案

日期：2026-05-27

## 1. 中配目标

高配方案 `STM32H7 + USB HS PHY + ESP32-C6 + 宽电压高速电平转换` 性能余量足，但成本和 PCB 难度明显偏高。中配版本的目标是：

| 项目 | 中配目标 |
|---|---|
| 成本 | 明显低于高配，尽量接近两块普通无线模块 + 高速 DAP 主控的成本 |
| 兼容 | 电脑端仍然枚举成标准 CMSIS-DAP + CDC 串口 |
| 下载速度 | 成熟固件目标 `300~600 KB/s`，条件好时冲 `800 KB/s` 左右 |
| 硬件余量 | 保留 USB High-Speed、SPI/SDIO 无线通道、NRST、SWO/RTS/CTS 焊盘 |
| 放弃项 | 不追求 1.2V 到 5V 全电平、不追求完整 JTAG、不追求 J-Link WiFi 级别 |

核心思路：用 `CH32V307` 替代 `STM32H7 + 外置 USB HS PHY`。CH32V307 自带 USB2.0 High-Speed 480Mbps PHY，价格低很多，适合做低成本 CMSIS-DAP v2 主控。

## 2. 推荐中配架构

```text
同一块 PCB，两块模块配对使用

电脑端角色：
PC / Keil / 串口助手
    │ USB-C, USB High-Speed
    ▼
CH32V307 主控
    ├── CMSIS-DAP v1/v2
    ├── USB CDC 虚拟串口
    └── 无线数据调度
          │ SPI 或 SDIO
          ▼
ESP32-C6 / ESP32-S3 无线协处理器
          │ Wi-Fi / ESP-NOW / UDP
          ▼
板端模块

板端角色：
ESP32-C6 / ESP32-S3 无线协处理器
          │ SPI 或 SDIO
          ▼
CH32V307 主控
    ├── 本地执行 SWD
    ├── 目标 UART 桥接
    ├── 控制 NRST
    └── 采样 VTREF
          │
          ▼
3.3V SWD/UART 保护与可选电平转换
          │
          ▼
目标 STM32 / Cortex-M MCU
```

## 3. 芯片选择

### 3.1 主控：CH32V307RCT6 / CH32V307VCT6

| 项目 | 说明 |
|---|---|
| CPU | RISC-V4F，最高 144MHz |
| 存储 | 常见型号 256KB Flash、64KB SRAM |
| USB | USB2.0 High-Speed Host/Device，内置 480Mbps PHY |
| 外设 | SPI、SDIO、UART、DMA、定时器、USB FS/HS |
| 成本 | 嘉立创/LCSC 常见价格约二十多元人民币级别 |
| 风险 | 生态不如 STM32H7，CMSIS-DAP v2 和 TinyUSB 适配工作量更大 |

选择它的原因是：它把高配里最贵、最难布线的 `STM32H7 + USB3320/USB3300` 组合压缩成一颗低价 MCU，同时保留 USB HS。

### 3.2 无线：ESP32-C6-WROOM-1U 或 ESP32-S3-WROOM-1U

| 方案 | 推荐度 | 说明 |
|---|---|---|
| ESP32-C6-WROOM-1U | 推荐 | 支持 Wi-Fi 6 2.4GHz，适合作为后续主力；建议外置天线版本 |
| ESP32-S3-WROOM-1U-N16R8 | 稳妥 | 资料更多、RAM/Flash 余量更大，做无线协处理器开发更舒服 |
| ESP32-C3-MINI-1 | 低成本 | 价格更低，但速度和余量更差，适合低配而不是中配 |

主控到 ESP 建议一版 PCB 同时预留：

| 连接 | 用途 |
|---|---|
| SPI | 第一版先跑 SPI，调试简单 |
| SDIO 4-bit | 后续固件提速用 |
| UART | 日志、烧录、救援升级 |
| EN/BOOT | 允许 CH32V307 控制 ESP 进入下载模式 |
| IRQ | ESP 通知 CH32V307 有新无线包 |

Espressif 的 ESP-Hosted 方案支持把 ESP 系列作为 Wi-Fi/BLE 协处理器，通过 SPI、SDIO 或 UART 与主控通信。中配版本不一定直接跑完整 ESP-Hosted 网络栈，也可以只借鉴它的传输结构，自己做更轻量的 DAP/UART 帧传输。

## 4. 中配硬件裁剪

### 4.1 保留

| 硬件 | 原因 |
|---|---|
| USB-C | 电脑端连接、供电、升级 |
| USB High-Speed | CH32V307 内置，成本不高，必须保留 |
| Wi-Fi 模组外置天线版本 | 无线烧录稳定性比省几元成本重要 |
| SWD：SWCLK/SWDIO/GND/VTREF/NRST | 下载稳定的核心接口 |
| UART：TX/RX/GND/VTREF | 无线串口 |
| 10Pin Cortex 焊盘 | 不一定焊座，但 PCB 上建议保留 |
| SWO、RTS、CTS 焊盘 | 后续调试增强和高速串口流控 |
| SPI + SDIO 双通道布线 | 软件先简单、后续可提速 |

### 4.2 裁剪

| 高配内容 | 中配处理 |
|---|---|
| STM32H743 | 换成 CH32V307 |
| USB3320/USB3300 外置 PHY | 删除，使用 CH32V307 内置 USB HS PHY |
| 1.1V 到 5.5V 全电平高速接口 | 改成主打 3.3V，预留可选电平转换版本 |
| 完整 JTAG | 第一版只做 SWD，JTAG 只预留焊盘 |
| 大容量外部 QSPI Flash | 可选，不作为必焊；ESP 模组本身已有 Flash |
| 电流检测芯片 | 可选，不作为中配标配 |

## 5. 接口和电平策略

中配版本建议明确写在产品说明里：**默认支持 3.3V SWD 和 3.3V UART，5V 引脚只作为电源，不代表 5V 逻辑输入输出。**

| 信号 | 中配做法 |
|---|---|
| SWCLK | CH32V307 GPIO 输出，经 22Ω 到 47Ω 串阻和 ESD 到目标 |
| SWDIO | CH32V307 GPIO 双向，经串阻和 ESD 到目标 |
| NRST | NMOS/开漏下拉，目标侧上拉 |
| TXD | CH32V307 输出到目标 RX，串阻保护 |
| RXD | 目标 TX 到 CH32V307 RX，串阻保护；如要支持 5V UART，增加分压或电平转换 |
| VTREF | 采样目标 3.3V，低于阈值时禁止驱动 SWD |
| 5V | 只做 VIN/可控输出，默认不与逻辑相连 |

如果想稍微多花一点钱但保留更多适配性，可以增加一个可选电平转换版本：

| 版本 | 电平支持 | 成本 |
|---|---|---|
| 中配基础版 | 3.3V SWD/UART | 最低 |
| 中配增强版 | 1.8V/2.5V/3.3V SWD，5V UART 输入保护 | 稍高 |
| 高配版 | 1.1V 到 5.5V SWD/JTAG/UART | 最高 |

## 6. BOM 粗估

下面是单个模块的粗略 BOM，不含外壳、包装、税费、研发摊销。两块模块成套使用，所以成套成本约为单个模块的两倍。

| 项目 | 基础中配 | 增强中配 |
|---|---:|---:|
| CH32V307 | ¥17~25 | ¥17~25 |
| ESP32-C6/S3 模组 | ¥18~35 | ¥18~35 |
| USB-C、ESD、晶振、按键、LED | ¥5~10 | ¥5~10 |
| 电源 LDO/DCDC、保护 | ¥5~12 | ¥6~15 |
| SWD/UART 保护 | ¥2~5 | ¥6~15 |
| 排针/连接器/天线座 | ¥4~10 | ¥5~12 |
| PCB 与贴片小批量摊销 | ¥15~35 | ¥18~40 |
| 单模块小批量成本 | ¥66~132 | ¥75~152 |
| 成套两模块小批量成本 | ¥132~264 | ¥150~304 |

量产后成本会下降，但无线烧录器天然需要两块完整模块，所以很难和一只 100 多元的有线高速 DAP 正面对比成本。有线 DAP 只有一块板，没有第二个无线端，也没有配对、无线稳定性、电池/远端供电这些成本。

## 7. 速度预期

| 阶段 | 预期速度 | 说明 |
|---|---:|---|
| V0：CMSIS-DAP v1 HID + SPI 无线 | `50~150 KB/s` | 先稳定跑通 |
| V1：CMSIS-DAP v2 bulk + SPI 无线 | `150~350 KB/s` | USB 不再是主要瓶颈 |
| V2：SDIO/高速 SPI + 批量 DAP | `300~600 KB/s` | 中配最现实的成熟目标 |
| V3：私有高速下载 + 本地 flash loader | `600~900 KB/s` | 需要配套上位机或 Keil 外插件思路 |

中配不要承诺 `1MB/s` 以上。不是完全做不到，而是性价比会快速下降，且受无线环境、目标 Flash、Keil 调用方式影响很大。

## 8. 和三个版本的对比

| 版本 | 架构 | 成本 | 速度 | 适合场景 |
|---|---|---:|---:|---|
| 低配 | ESP32-S3 单芯片 x2 | 最低 | `50~200 KB/s` | 爱好者、教学、低成本无线串口 + 基础下载 |
| 中配推荐 | CH32V307 + ESP32-C6/S3 x2 | 中等 | `300~600 KB/s` | 你现在更适合的方案 |
| 高配 | STM32H7 + USB HS PHY + ESP32-C6 x2 | 高 | `800KB/s~1MB/s+` | 小批量产品、追求接近商业无线调试器 |

## 9. 最终建议

如果你觉得高配太贵，我建议直接改成：

| 决策 | 中配推荐 |
|---|---|
| 主控 | CH32V307RCT6，成本低且自带 USB HS |
| 无线 | ESP32-C6-WROOM-1U；若开发稳妥优先，用 ESP32-S3-WROOM-1U-N16R8 |
| 接口 | 只正式支持 SWD，不做完整 JTAG |
| 电平 | 基础版主打 3.3V；增强版再加 1.8V/2.5V/5V 保护 |
| USB | CMSIS-DAP v1/v2 + CDC + DFU |
| 无线链路 | 第一版 SPI，PCB 预留 SDIO |
| 成套售价目标 | 基础版尽量压到 ¥199~299 区间；增强版 ¥299~399 更现实 |

这个中配方案的性价比更平衡：它不像 ESP32-S3 单芯片那样太快撞到 USB 和调度瓶颈，也不像 STM32H7 高配那样成本压不下来。真正要控制成本，最重要的是接受一个边界：**中配只主打 3.3V SWD + 无线串口，不承诺全电平、全 JTAG 和 1MB/s 级别下载。**
