# ESP32-C6 通讯协处理器 PCB 器件与布局设计建议

日期：2026-05-28

## 1. 直接结论

本项目里 `ESP32-C6` 建议只作为通讯协处理器使用，主控仍负责 DAP/SWD/JTAG/USB 高速链路和目标板时序。第一版硬件优先使用 `ESP32-C6-WROOM-1U` 或 `ESP32-C6-WROOM-1` 模组，不建议第一版直接使用裸 `ESP32-C6` 芯片。

| 项目 | 建议 | 原因 |
|---|---|---|
| 推荐封装形态 | 优先 `ESP32-C6-WROOM-1U-N8`，备选 `ESP32-C6-WROOM-1-N8` | 模组已经集成 40 MHz 晶振、SPI Flash、射频匹配和屏蔽罩，能显著降低 RF 和晶振风险 |
| 天线方式 | USB 小外壳、金属外壳、靠近电脑 USB 口时优先 `WROOM-1U` 外置天线 | 外置天线更容易避开 USB 口、外壳、地平面和手持遮挡 |
| PCB 层数 | 推荐 4 层板 | ESP32-C6 有 RF、USB FS、SDIO/SPI 高速线，4 层更容易保证完整地平面、阻抗和回流路径 |
| 晶振处理 | 用模组时不需要外部 40 MHz 晶振布局；裸芯片时必须重点处理 | 模组内部已处理主晶振；裸芯片需要 40 MHz 晶振、负载电容、串联器件、地隔离和频偏调试 |
| 主控接口 | 推荐预留 `SDIO + SPI + UART0`，第一版至少焊出 SPI/UART0 | SDIO 适合后期提速，SPI 适合快速打通，UART0 适合下载和日志兜底 |

## 2. 方案边界

| 边界 | 本文默认假设 |
|---|---|
| ESP32-C6 职责 | Wi-Fi / BLE / 802.15.4 通讯、无线数据打包、OTA、配置页面、与主控交换数据 |
| 主控职责 | USB CMSIS-DAP、SWD/JTAG 物理时序、目标串口、目标复位、电平转换、目标电压检测 |
| 供电来源 | 整板 5 V 输入，经 3.3 V 电源给主控和 ESP32-C6；ESP32-C6 侧单独保证瞬态电流余量 |
| 第一版目标 | 优先稳定可调试，不追求极限尺寸，不把 RF 裸芯片设计放到第一版风险里 |

## 3. 推荐器件清单

### 3.1 ESP32-C6 模组方案的最小可打样 BOM

| 模块 | 器件/电路 | 推荐规格/数值 | 数量 | 说明 |
|---|---|---:|---:|---|
| 通讯模组 | `ESP32-C6-WROOM-1U-N8` | 8 MB Flash，外接天线座 | 1 | 推荐首选；外置天线更适合无线 DAP 这种靠近电脑和目标板的设备 |
| 通讯模组备选 | `ESP32-C6-WROOM-1-N8` | 8 MB Flash，板载 PCB 天线 | 1 | 适合塑料外壳、天线能伸出板边、周围没有金属遮挡的版本 |
| 3.3 V 电源 | LDO 或 DC/DC | 给 ESP32-C6 预留不小于 500 mA 能力 | 1 | 若整板还有 CH32/STM32 和目标侧电路，建议 3.3 V 总能力做到 800 mA~1 A 以上 |
| 电源入口电容 | MLCC | 10 uF / 6.3 V 或更高耐压 | 1~2 | 放在 ESP32-C6 3V3 入口附近，用于承受 Wi-Fi 发射瞬态电流 |
| 高频去耦 | MLCC | 0.1 uF + 1 uF | 各 1~2 | 靠近模组 3V3/GND 引脚，走线短，地端就近打孔 |
| EN 复位 RC | 电阻 + 电容 | 10 k 上拉到 3.3 V，1 uF 到 GND | 1 组 | 官方推荐 EN/CHIP_PU 做 RC 延时，避免电源未稳定时启动失败 |
| 复位按键 | 轻触开关 | EN 到 GND | 1 | 调试必备；也可由主控 GPIO 或开漏管脚拉低 EN |
| 下载/BOOT | 轻触开关或测试点 | IO9 到 GND，IO9 默认上拉 | 1 | ESP32-C6 下载模式会用到 IO9；不要在复位期间用外部电路强拉异常启动状态 |
| USB-C 调试口 | USB-C 母座 | USB 2.0 FS 即可 | 0~1 | 建议保留给 ESP32-C6 固件下载、日志和救砖；量产版可不焊 |
| USB CC 电阻 | 电阻 | CC1/CC2 各 5.1 k 下拉 | 2 | 当 USB-C 作为设备口时必须放置 |
| USB 串联电阻 | 电阻 | D+/D- 各 22 Ω 或 33 Ω | 2 | 靠近 ESP32-C6 侧放置；用于阻尼和 EMI 调整 |
| USB 预留电容 | MLCC 焊盘 | D+/D- 到 GND，默认不焊 | 2 | 官方建议预留；只有 EMI 调试需要时再焊小电容 |
| USB ESD | 低电容 TVS | 适合 USB 2.0 FS | 1 | 靠近 USB 连接器放，ESD 地路径必须短 |
| UART0 调试口 | 排针/焊盘 | TXD0、RXD0、GND、3V3 | 1 组 | 串口日志、下载兜底；TXD0 建议串 499 Ω 或预留 0 Ω/499 Ω |
| 主控通信接口 | SDIO | IO18 CMD、IO19 CLK、IO20~IO23 DATA0~3 | 1 组 | 推荐预留，适合后期高吞吐 |
| SDIO 上拉 | 电阻 | CMD/DATA0~3 上拉 10 k~47 k | 5 | 按主控和速率调试，第一版建议留焊盘 |
| SDIO 串阻 | 电阻焊盘 | 每线 0 Ω/22 Ω/33 Ω 可选 | 6 | 靠近驱动端或 ESP 侧预留，用于过冲和 EMI 调整 |
| 主控通信接口 | SPI | SCLK、MOSI、MISO、CS、IRQ/READY | 1 组 | 第一版最容易打通；SCLK 必须预留串阻 |
| 状态指示 | LED + 电阻 | 电源、无线连接、数据活动 | 2~3 | LED 不要使用启动绑带管脚，或必须保证复位期间电平正确 |
| 测试点 | 焊盘 | 3V3、GND、EN、IO9、TXD0/RXD0、SDIO/SPI 关键线 | 若干 | 第一版调试价值很高，后期可删减 |

### 3.2 如果坚持使用裸 ESP32-C6 芯片，需要额外增加的器件

裸芯片方案可以更小或更低 BOM 成本，但第一版不推荐。它会把晶振、Flash、电源滤波、RF 匹配、天线调试和阻抗控制全部变成你的责任。

| 子系统 | 需要增加的器件 | 关键要求 |
|---|---|---|
| 主晶振 | 40 MHz 晶振、两颗负载电容、XTAL_P 串联器件 | ESP32-C6 固件只支持 40 MHz 主晶振；晶振精度建议控制在 ±10 ppm；XTAL_P 官方初始建议串 24 nH，再按整机 RF 测试调整 |
| RTC 晶振 | 32.768 kHz 晶振、负载电容、可选并联偏置电阻 | 可选；用于低功耗计时，ESR 要满足官方要求；无线 DAP 如果不追求深睡眠可先不焊 |
| Flash | QSPI Flash 或选择带封装内 Flash 的型号 | 裸 QFN40 通常需要外部 Flash；SPI 线上预留 0 Ω 串阻，便于调驱动强度和 EMI |
| RF 匹配 | 靠近芯片的 CLCCL 匹配、靠近天线的 CLC 匹配 | RF 走线必须 50 Ω；0201 器件更容易贴近射频脚但焊接难度更高 |
| 天线/连接器 | PCB 天线、陶瓷天线或 IPEX/MHF 连接器 | 需要实测调匹配；没有仪器时风险很高 |
| 模拟/RF 供电滤波 | LC/CLC 滤波、10 uF、1 uF、0.1 uF | RF 电源和普通数字电源不要随意共线，去耦必须贴近引脚 |
| 底部焊盘 | EPAD 接地过孔阵列 | 裸芯片底部焊盘至少通过多个地孔接到完整地平面，兼顾散热和 RF 回流 |

## 4. 原理图设计要点

### 4.1 供电

| 项目 | 要求 |
|---|---|
| ESP32-C6 供电电压 | 3.0 V~3.6 V，推荐 3.3 V |
| 电流能力 | ESP32-C6 单独预留不小于 500 mA；整板 3.3 V 电源按主控、ESP、LED、外设总和再留余量 |
| 去耦位置 | 10 uF 放在 ESP32-C6 3V3 入口附近，0.1 uF/1 uF 靠近模组电源脚 |
| 地回路 | 电容地端就近打到 GND 面，不要用细长地线绕回 |
| 电源分区 | DC/DC 电感、开关节点远离 ESP32-C6 天线、RF 区、裸芯片晶振区 |

### 4.2 复位和启动

| 信号 | 设计建议 | 注意事项 |
|---|---|---|
| EN/CHIP_PU | 10 k 上拉 + 1 uF 到 GND；按键或主控可拉低复位 | EN 不能悬空，复位线尽量短，避免外部干扰导致随机重启 |
| IO9 | 默认上拉，预留 BOOT 按键到 GND | 官方建议 IO9 上拉，不要在 IO9 加大电容，否则可能误入下载模式 |
| IO8 | 避免复位期间被外设强拉到错误状态 | IO8/IO9 共同影响启动模式；使用这些脚做业务 IO 时必须核查上电默认电平 |
| 主控控制下载 | 主控可以控制 EN 和 IO9 | 建议通过 0 Ω 电阻或焊盘跳线连接，第一版便于隔离问题 |

### 4.3 主控到 ESP32-C6 的通信接口

| 接口 | 推荐用途 | PCB 要求 | 第一版建议 |
|---|---|---|---|
| SDIO | 高吞吐无线数据通道 | IO18~IO23；50 Ω 单端；CMD/DATA 相对 CLK 长度差控制在 ±50 mil；总长度尽量 < 2000 mil | 必须预留焊盘和串阻，固件可后期启用 |
| SPI | 中速稳定通道，开发难度低 | SCLK 旁边包地，所有线预留 0 Ω/22 Ω 串阻；长度尽量短 | 第一版主力打通接口 |
| UART0 | 下载、日志、低速兜底 | TXD0/RXD0 短走线；TXD0 串阻靠近 ESP32-C6 | 必须引出到焊盘或排针 |
| USB Serial/JTAG | ESP32-C6 本身下载、日志、JTAG | D-/D+ 分别为 IO12/IO13；90 Ω 差分；串阻靠近 ESP32-C6 | 建议保留调试口或至少测试点 |

## 5. PCB 层数和叠层建议

### 5.1 推荐层数

| 方案 | 是否推荐 | 适用条件 | 风险 |
|---|---|---|---|
| 4 层板 | 强烈推荐 | 无线 DAP 正式第一版、中高速 SPI/SDIO、USB 调试口、紧凑布局 | 成本略高，但大幅降低返板概率 |
| 2 层板 | 仅低成本验证可用 | 使用 WROOM 模组、只跑低速 SPI/UART、板子足够大、天线净空充足 | 地平面容易被割裂，USB/SDIO/RF 回流难保证 |
| 裸芯片 2 层板 | 不建议 | 除非只是非无线实验板 | RF、晶振、电源和地回流风险过高 |

### 5.2 4 层板推荐叠层

| 层 | 用途 | 规则 |
|---|---|---|
| L1 Top | 器件、短高速线、RF/USB/SDIO 关键线 | RF、USB、SDIO 尽量在同层完成，减少过孔 |
| L2 GND | 完整地平面 | 不走信号，不切割；给 RF、USB、SDIO、晶振提供连续参考面 |
| L3 Power/Slow | 3.3 V、电源分配、少量低速信号 | RF 和晶振下方仍优先保持完整 GND 参考，不要用电源面割裂回流 |
| L4 Bottom | 低速信号、少量辅助线 | 不建议放高噪声器件；底层走线也要避免穿过天线净空 |

## 6. 布局布线重点

### 6.1 模组摆放和天线

| 项目 | 要求 |
|---|---|
| WROOM-1 板载天线 | 模组天线端尽量伸出主板边缘，天线下方和前方不要铺铜、不要走线、不要放器件 |
| 板载天线净空 | 如果天线不能伸出板边，天线区域周围至少留 15 mm 净空，净空内不铺铜、不走线、不放器件；有条件时直接挖掉基板 |
| WROOM-1U 外接天线 | 模组本体没有板载天线禁布区，但外接天线必须远离 USB 口、金属外壳、大电流线、电感和高速线 |
| 外壳影响 | 最终外壳、USB 插入方向、手持位置都会影响天线，打样后必须实测吞吐和距离 |
| 分区 | ESP32-C6 天线/RF 区放板边，DC/DC 电感、USB 连接器、主控晶振、目标接口尽量远离天线 |

### 6.2 RF 规则

使用 WROOM 模组时，大部分 RF 匹配已经在模组内部完成，主板重点是天线净空和外接天线摆放。使用裸芯片时必须执行以下规则。

| 项目 | 裸芯片设计要求 |
|---|---|
| 阻抗 | RF 走线 50 Ω，按板厂叠层计算线宽 |
| 走线 | RF 线尽量短、等宽、不分叉、外层走线、不换层、不打过孔 |
| 拐角 | 用 135° 或圆弧，不用直角 |
| 参考面 | RF 线下方相邻层必须是连续 GND，不在 RF 线下方穿越其他信号 |
| 保护地 | RF 线两侧密集地孔围栏，但不要破坏阻抗线宽和天线净空 |
| 匹配 | 芯片侧 CLCCL 匹配靠近 RF 引脚；天线侧预留 CLC 匹配 |
| ESD | 天线口建议加低寄生 ESD，器件接地路径要短 |

### 6.3 USB 规则

| 项目 | 要求 |
|---|---|
| 引脚 | ESP32-C6 USB_D- 为 IO12，USB_D+ 为 IO13 |
| 差分阻抗 | 90 Ω ±10% |
| 等长 | D+/D- 平行、等长、少过孔 |
| 串阻 | 22 Ω 或 33 Ω 靠近 ESP32-C6 放 |
| ESD | TVS 靠近 USB 连接器放，地回路短而宽 |
| 参考面 | USB 下方必须有连续 GND；如必须换层，每个换层点旁加成对地回流孔 |

### 6.4 SDIO/SPI 规则

| 项目 | SDIO | SPI |
|---|---|---|
| 主要信号 | IO18 CMD、IO19 CLK、IO20 DATA0、IO21 DATA1、IO22 DATA2、IO23 DATA3 | SCLK、MOSI、MISO、CS、IRQ/READY |
| 阻抗 | 50 Ω 单端，±10% | 建议按 50 Ω 单端处理，至少保证连续参考地 |
| 长度 | CMD/DATA 相对 CLK 长度差 ±50 mil 内，总长度尽量 < 2000 mil | SCLK 最短且旁边包地，数据线长度不要明显超长 |
| 换层 | 尽量不换层 | 尽量不换层；如换层，加地回流孔 |
| 串阻 | 每线预留 0 Ω/22 Ω/33 Ω | SCLK 必留，数据线建议预留 |
| 上拉 | CMD/DATA0~3 加 10 k~47 k 上拉焊盘 | MISO/MOSI/CS 视主控默认状态加上拉/下拉 |

## 7. 晶振周围是否需要特殊处理

### 7.1 使用 ESP32-C6-WROOM 模组时

模组内部已经有 40 MHz 晶振、Flash 和 RF 匹配。主板上不需要再放 ESP32-C6 的 40 MHz 主晶振，也不需要处理主晶振周围铺铜。你需要做的是：

| 项目 | 处理 |
|---|---|
| 模组下方 | 按官方封装焊盘和禁布区做，不要自行改封装 |
| 天线区域 | WROOM-1 必须留天线净空；WROOM-1U 重点处理外接天线位置 |
| 主控晶振 | 如果主控另有晶振，主控晶振不要靠近 ESP32-C6 天线和 RF 区 |

### 7.2 使用裸 ESP32-C6 芯片时

晶振周围确实要做特殊处理，但不是简单地“所有层都不铺铜”。正确做法如下。

| 项目 | 正确做法 | 原因 |
|---|---|---|
| 顶层铺铜 | 如果顶层有足够 GND，晶振本体周围建议做局部 keepout 隔离；如果顶层 GND 很少，不要强行挖孤岛 | 降低寄生电容和热传导对频偏的影响 |
| 内层地 | 晶振、RF、芯片下方必须有完整 GND 参考面 | 晶振和 RF 都需要稳定回流路径，不能把所有层挖空 |
| 晶振走线 | XTAL_P/XTAL_N 不打过孔，不穿层，尽量短且受控 | 过孔和长线会增加寄生参数和干扰耦合 |
| 下方布线 | 晶振下方不要走高速数字线，最好不走任何信号线 | SDIO、USB、UART 等高频分量会串扰晶振 |
| 地孔围栏 | 晶振走线两侧可加密地孔隔离 | 降低外部信号耦合 |
| 负载电容 | 放在晶振两侧，地端靠近晶振地和完整 GND | 负载电容决定实际振荡频率，需要可调试 |
| 串联器件 | XTAL_P 串联器件靠近芯片侧，初始可参考 24 nH | 用于抑制晶振高次谐波对 RF 性能的影响，最终按实测调整 |
| 周边器件 | 晶振附近不要放电感、天线、大电流开关节点 | 避免磁场、热和 RF 耦合导致频偏或灵敏度下降 |

## 8. 推荐第一版接口预留

| 预留项 | 是否建议 | 目的 |
|---|---|---|
| ESP32-C6 USB-C 或 USB 测试焊盘 | 建议 | 固件下载、日志、救砖、USB Serial/JTAG |
| UART0 四针焊盘 | 必须 | 最低成本下载和日志 |
| EN + IO9 测试点 | 必须 | 复位、进下载模式、排查启动失败 |
| SDIO 全 6 线焊盘和串阻 | 必须 | 后续无线吞吐升级 |
| SPI 4 线 + IRQ/READY | 必须 | 第一版快速打通主控通信 |
| 外接天线接口空间 | 使用 WROOM-1U 时必须 | 方便换不同天线、调整摆放 |
| 电流测量跳线 | 建议 | 用 0 Ω 或电流采样焊盘隔开 ESP32-C6 3V3 支路，便于测 Wi-Fi 峰值电流 |

## 9. PCB 打样检查表

| 检查项 | 通过标准 |
|---|---|
| ESP32-C6 3.3 V 余量 | 单独预留 >= 500 mA，Wi-Fi 发射时 3.3 V 不明显跌落 |
| EN 复位 | EN 有 10 k/1 uF RC，按键和主控拉低都能可靠复位 |
| BOOT | IO9 默认上拉，按住 BOOT 再复位能进入下载模式 |
| USB | D+/D- 有 22 Ω/33 Ω 串阻，差分 90 Ω，ESD 靠近连接器 |
| SDIO | IO18~IO23 走线短，等长满足 ±50 mil，串阻和上拉焊盘齐全 |
| SPI | SCLK 串阻已预留，CLK 周围有完整参考地 |
| 天线净空 | WROOM-1 天线区不铺铜、不走线、不放器件；WROOM-1U 外接天线远离金属和高噪声区 |
| 地平面 | 4 层板 L2 为完整 GND，不被电源或信号割裂 |
| 电源电容 | ESP32-C6 3V3 入口有 10 uF，近端有 0.1 uF/1 uF |
| 测试点 | 3V3、GND、EN、IO9、TXD0、RXD0、SPI/SDIO 关键线都有可测点 |
| 热和噪声 | DC/DC 电感、USB 口、主控晶振远离 ESP32-C6 天线和裸芯片晶振区 |

## 10. 建议的最终取舍

| 目标 | 建议选型 |
|---|---|
| 第一版尽快稳定通信 | `ESP32-C6-WROOM-1U-N8 + SPI 主通道 + UART0/USB 调试 + SDIO 预留` |
| 后续追求无线吞吐 | 启用 `SDIO`，保留 SPI 作为兼容/调试通道 |
| 成本优先且外壳为塑料 | `ESP32-C6-WROOM-1-N4/N8`，但必须满足天线净空 |
| 体积极限或量产降本 | 第二版以后再评估裸 `ESP32-C6`，前提是有 RF 调试和频偏测试条件 |
