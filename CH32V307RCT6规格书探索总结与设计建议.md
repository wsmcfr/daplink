# CH32V307RCT6 规格书探索总结与设计建议

日期：2026-05-27

## 1. 文档目的

本文记录对 `C2988083_单片机(MCU-MPU-SOC)_CH32V307RCT6_规格书_WJ1622413.PDF` 的本轮阅读结论，并把结论转化为无线 DAP 烧录器中配方案的硬件和固件设计约束。

本轮重点不是泛泛介绍 CH32V307，而是回答一个工程问题：`CH32V307RCT6` 是否适合作为 `CH32V307 + ESP32-C6/S3` 中配无线 DAP 的主控，以及后续原理图、PCB 和固件设计应该怎么做。

## 3. 芯片总体判断

| 结论 | 说明 |
|---|---|
| 适合继续作为中配主控 | `CH32V307RCT6` 集成 USB2.0 High-Speed PHY，省掉高配方案里的外置 USB HS PHY 和更贵主控。 |
| 性能余量基本够用 | 144MHz RISC-V4F、DMA、SPI、SDIO、多 UART 能支撑 `CMSIS-DAP v2 + CDC + 本地 SWD 执行 + ESP 无线协处理`。 |
| 软件生态是主要风险 | WCH 生态、USB HS 设备栈、CMSIS-DAP v2 和 TinyUSB 适配成熟度不如 STM32，需要先做验证板或最小原型验证。 |
| 不适合一开始堆满功能 | 第一版不要同时追求 MSC 拖拽烧录、完整 JTAG、全电平适配、SDIO 高速链路和私有高速下载。 |

本项目中应把 `CH32V307RCT6` 定位为：低成本 USB HS + 本地 SWD 执行主控，而不是高端全功能调试器主控。

## 4. 关键资源总结

| 类别 | 手册结论 | 对无线 DAP 的意义 |
|---|---|---|
| CPU | 青稞 `RISC-V4F`，最高 `144MHz` | 足够处理 USB、无线帧调度、DAP 命令解析和 GPIO SWD 时序。 |
| 中断 | PFIC，支持快速中断和硬件中断堆栈 | 有利于 USB、SPI/SDIO、UART 并发调度。 |
| Flash/SRAM | `CH32V307RCT6` 常规资源为 `256KB Flash + 64KB SRAM` | 第一版 DAP/CDC/无线调度够用；复杂文件系统和大缓存要谨慎。 |
| USB HS | USB2.0 高速主机/设备，内置 `480Mbps PHY` | 电脑端可走 CMSIS-DAP v2 Bulk，USB 不应成为主要瓶颈。 |
| USB FS/OTG | 另有全速/OTG 控制器 | 可作为备选 USB 或调试验证通道，但中配方案优先使用 USB HS。 |
| SPI | 最高 3 组 SPI，主/从最高 `72MHz` | 第一版建议用 SPI 接 ESP，开发和调试成本低。 |
| SDIO | 支持 1-bit、4-bit、8-bit，最高 `48MHz` | 后续提速可用 4-bit SDIO 接 ESP，但引脚占用和驱动复杂度更高。 |
| UART | 多组 USART/UART | 可同时支持目标 UART、ESP 日志/下载、内部调试串口。 |
| GPIO | 最多 80 个 IO，RCT6 引出约 51 个 GPIO | RCT6 引脚够用，但高速 USB、SDIO、SPI、调试口会吃掉关键脚。 |
| DMA | 2 组 DMA，共 18 通道 | USB、SPI、SDIO、UART 都应优先用 DMA 降低 CPU 占用。 |
| RTC/VBAT | 有 RTC 和后备域 | 本项目第一版不是重点，可只按规范接 VBAT 或并到 3.3V。 |

## 5. CH32V307RCT6 核心引脚约束

下面是后续原理图必须固定或优先保留的关键引脚。

| 功能 | 引脚 | 设计要求 |
|---|---|---|
| USB HS DM | `PB6 / USBHS_DM` | 电脑端 USB HS 差分线之一，禁止复用给普通 GPIO。 |
| USB HS DP | `PB7 / USBHS_DP` | 电脑端 USB HS 差分线之一，禁止复用给普通 GPIO。 |
| USB FS/OTG DM | `PA11 / OTG_FS_DM` | 可作为备用 USB FS，不建议和主 USB HS 混用到同一接口。 |
| USB FS/OTG DP | `PA12 / OTG_FS_DP` | 可作为备用 USB FS，不建议和主 USB HS 混用到同一接口。 |
| CH32 调试数据 | `PA13 / SWDIO` | 必须保留给 WCH-Link 调试下载，不要拿去做目标 SWD。 |
| CH32 调试时钟 | `PA14 / SWCLK` | 必须保留给 WCH-Link 调试下载，不要拿去做目标 SWD。 |
| 复位 | `NRST` | 建议留调试排针、复位按键和 RC 复位电路。 |
| 启动选择 | `BOOT0` | 默认下拉到 GND，预留按键或焊盘进入系统 BootLoader。 |
| 启动选择 | `PB2 / BOOT1` | 不要悬空；若不用，按输入下拉处理，减少低功耗异常电流。 |
| HSE 输入 | `OSC_IN / PD0` | 建议接外部 8MHz 晶体或有源时钟，支持 USB/PLL 稳定工作。 |
| HSE 输出 | `OSC_OUT / PD1` | 和 HSE 输入配套使用，靠近芯片布局。 |
| LSE | `PC14/PC15` | 第一版可不焊 32.768kHz；如不用，按手册建议处理。 |

重要判断：`PA13/PA14` 是 CH32 自身调试口，不应该作为目标板 SWD 接口。目标板 SWD 应使用另外的 GPIO 通过固件模拟或专用时序实现。

## 6. 建议的一版引脚分配

这是一版原理图的候选分配，目标是先跑通，不追求最终极限性能。

| 模块 | 信号 | 建议 CH32 引脚 | 说明 |
|---|---|---|---|
| USB-C | USBHS_DM | `PB6` | 走 USB HS 差分线，靠近接口放低电容 ESD。 |
| USB-C | USBHS_DP | `PB7` | 和 `PB6` 等长、等阻抗、短走线。 |
| CH32 调试口 | SWDIO | `PA13` | 只接 WCH-Link 调试排针。 |
| CH32 调试口 | SWCLK | `PA14` | 只接 WCH-Link 调试排针。 |
| CH32 调试口 | NRST | `NRST` | 接调试排针和复位按键。 |
| ESP SPI | SPI2_NSS | `PB12` | 第一版主控到 ESP 的片选。 |
| ESP SPI | SPI2_SCK | `PB13` | 第一版主控到 ESP 的 SPI 时钟。 |
| ESP SPI | SPI2_MISO | `PB14` | 同时也是 SDIO_D0 备选脚，后续使用 SDIO 时需重新评估。 |
| ESP SPI | SPI2_MOSI | `PB15` | 同时也是 SDIO_D1 备选脚，后续使用 SDIO 时需重新评估。 |
| ESP 控制 | ESP_EN | `PC0` 或其他空闲 GPIO | 控制 ESP 复位/使能。 |
| ESP 控制 | ESP_BOOT | `PC1` 或其他空闲 GPIO | 控制 ESP 进入下载模式。 |
| ESP 中断 | ESP_IRQ | `PC2` 或其他 EXTI GPIO | ESP 通知 CH32 有新无线包。 |
| ESP UART | USART2_TX | `PA2` | CH32 到 ESP 日志/下载串口。 |
| ESP UART | USART2_RX | `PA3` | ESP 到 CH32 日志/下载串口。 |
| 目标 SWD | SWCLK_OUT | `PA8` | 目标侧 SWCLK，串 22R 到 47R，靠近驱动端。 |
| 目标 SWD | SWDIO_IO | `PA9` | 目标侧 SWDIO，需支持方向切换。 |
| 目标 SWD | TARGET_NRST | `PA10` | 通过 NMOS/开漏方式拉低目标复位。 |
| 目标 UART | TARGET_TX | `PB10 / USART3_TX` | 模块发送到目标 RX。 |
| 目标 UART | TARGET_RX | `PB11 / USART3_RX` | 目标 TX 进入模块 RX。 |
| VTREF | ADC 输入 | `PA0` 或 `PA1` | 分压采样目标电压，未检测到目标电压时禁止驱动 SWD。 |
| 状态灯 | LED | 避开 `PC13/PC14/PC15` | 备份域 IO 驱动能力有限，不建议驱动 LED。 |

如果一版 PCB 空间允许，应把 SDIO 4-bit 也以焊盘形式预留出来。

| SDIO 信号 | 默认候选引脚 | 注意事项 |
|---|---|---|
| SDIO_D0 | `PC8` 或条件映射 `PB14` | 与以太网/批号/寄存器配置存在映射细节，原理图需留选择余地。 |
| SDIO_D1 | `PC9` 或条件映射 `PB15` | 与以太网/批号/寄存器配置存在映射细节，原理图需留选择余地。 |
| SDIO_D2 | `PC10` | 同时可复用 UART4/SPI3/DVP。 |
| SDIO_D3 | `PC11` | 同时可复用 UART4/SPI3/DVP。 |
| SDIO_CK | `PC12` | 高速时钟线，走线要短。 |
| SDIO_CMD | `PD2` | 需要上拉和 ESD 评估。 |

## 7. 电源设计约束

| 电源项 | 手册事实 | 设计建议 |
|---|---|---|
| VDD | 标准工作电压通常 `2.4~3.6V`，使用 USB 时要求 `3.0~3.6V` | 本项目统一按 `3.3V` 设计，不做低压主控运行。 |
| VIO | 大部分 IO 电源，且不能高于 VDD | `VIO` 和 `VDD` 同接 `3.3V`，降低电平域复杂度。 |
| VDDA | 必须与 VIO 相同，ADC 使用时不低于 `2.4V` | 接 `3.3V`，通过磁珠或 0R + 去耦隔离模拟噪声。 |
| VBAT | `1.8~3.6V` 后备电源 | 第一版可接 3.3V 或预留电池焊盘；连线尽量短。 |
| USB 工作电压 | USB 模块工作电压 `3.0~3.6V` | USB 插入时 3.3V 稳压必须稳定，避免枚举异常。 |
| 电源总电流 | 电源/地线绝对最大总电流 150mA 是芯片内部限制，不是整板电源预算 | CH32 自身功耗不高，但 ESP 发射峰值大，3.3V 稳压器建议按 `700mA~1A` 选型。 |

去耦建议：

| 位置 | 建议 |
|---|---|
| 每个 VDD/VIO | 每个电源脚附近放 `0.1uF`，走线尽量短。 |
| 主 3.3V | CH32 附近放 `4.7uF~10uF`。 |
| VDDA | `0.1uF + 1uF` 靠近引脚，模拟地回流不要绕高速 USB/SPI。 |
| USB-C VBUS | 加 TVS、保险丝或限流开关、输入电容。 |
| ESP 模组电源 | 模组附近单独放足够 bulk 电容，避免 Wi-Fi 峰值拉低 CH32 电源。 |

## 8. 时钟设计约束

| 项目 | 手册事实 | 设计建议 |
|---|---|---|
| HSI | 内置 8MHz RC，精度受温度影响 | 可用于启动和普通运行，不建议作为 USB 高速长期精确基准的唯一依据。 |
| HSE | 外部高速时钟支持 `3~25MHz` | 建议使用 `8MHz` 晶体或稳定有源时钟。 |
| USB 频率 | 使用 USB 功能时 CPU 频率需为 `48MHz/96MHz/144MHz` 等合法配置 | 中配方案优先跑 `144MHz`，保证 USB、SPI、SWD 调度余量。 |
| PLL | 主 PLL 输出最高 `144MHz` | 固件需按 WCH 推荐时钟树配置，避免 USB 枚举不稳定。 |
| HSE 晶体 | 8MHz 晶体典型启动时间约 `1.5~4ms` | 原理图按晶体厂商负载电容计算，不要直接照抄固定电容值。 |

## 9. USB 设计约束

| 项目 | 设计建议 |
|---|---|
| 接口角色 | 电脑端模块使用 USB Device，枚举 CMSIS-DAP v2 Bulk + CDC。 |
| 物理接口 | USB-C 只做 UFP 设备，`CC1/CC2` 各接 `5.1k` 下拉。 |
| 高速线 | `PB6/PB7` 到 USB-C 之间按 USB HS 差分线处理，避免过孔、长支线和不连续参考地。 |
| ESD | USB D+/D- 使用低电容 ESD，靠近 USB-C 放置。 |
| VBUS 检测 | 需要在固件或硬件上知道 USB 是否接入，可预留分压检测脚。 |
| 不建议 | 不要把 USB HS 和 USB FS 同时并到一个 USB-C D+/D-。 |

固件风险：必须优先验证 WCH USB HS 设备例程能稳定枚举，并确认 CMSIS-DAP v2 的 WinUSB/Bulk 端点配置是否可行。

## 10. 目标 SWD 与电平策略

本项目中配版应明确：默认只正式支持 `3.3V SWD` 和 `3.3V UART`，5V 只作为电源，不代表 5V 逻辑兼容。

| 信号 | 推荐电路 |
|---|---|
| SWCLK | CH32 GPIO 输出，经 `22R~47R` 串阻和低电容 ESD 到目标。 |
| SWDIO | CH32 GPIO 双向，经串阻和 ESD 到目标；如加电平转换，需固件控制方向。 |
| NRST | NMOS/开漏下拉，只拉低不强推高，目标侧由目标板上拉。 |
| VTREF | 分压进 ADC 或比较器，低于阈值时固件禁止驱动 SWD。 |
| UART_TX | CH32 输出到目标 RX，经串阻保护。 |
| UART_RX | 目标 TX 到 CH32 RX，基础版只接受 3.3V；若要兼容 5V UART，需要分压或电平转换。 |

不建议第一版使用自动方向电平转换器处理高速 SWDIO。自动方向器件容易在 SWD 半双工方向切换、弱上拉和高速边沿下出现不稳定。若要做增强版，优先考虑方向控制型双电源收发器，并由固件明确控制 `DIR/OE`。

## 11. 固件设计建议

| 固件模块 | 建议 |
|---|---|
| USB 栈 | 先用 WCH 官方 USB HS 例程跑通，再评估 TinyUSB 移植；不要一开始就重写底层 USB。 |
| CMSIS-DAP | 先实现最小 `DAP_Info`、`DAP_Connect`、`DAP_SWJ_Clock`、`DAP_Transfer`，后续再补 SWO、JTAG、Vendor 命令。 |
| DAP 版本 | 第一阶段可先 HID/v1 验证链路，正式性能目标应转 CMSIS-DAP v2 Bulk。 |
| SWD 引擎 | 板端 CH32 本地执行 SWD 时序，无线链路只传 DAP 命令和响应。 |
| 无线链路 | 第一版 CH32 与 ESP 使用 SPI + DMA + IRQ，先稳定，再追求 SDIO。 |
| 缓冲策略 | USB、无线、目标 SWD、CDC UART 之间必须有环形缓冲和超时控制。 |
| 错误恢复 | 必须处理无线丢包、目标 WAIT/FAULT、USB 断开、ESP 重启、VTREF 掉电。 |
| 看门狗 | 量产固件应启用 IWDG，但调试早期可先关闭或放宽超时。 |
| BootLoader | BOOT0 进系统 BootLoader；应用层可预留 USB/串口升级入口。 |

## 13. 对当前中配方案的修正建议

| 原方案点 | 本轮修正 |
|---|---|
| `CH32V307` 作为主控 | 继续保留，结论成立。 |
| `USB High-Speed` 必须保留 | 继续保留，但明确使用 `PB6/PB7`，按 USB HS PCB 规范处理。 |
| `SPI + SDIO 双通道预留` | 继续保留，但 SDIO 与 `PB14/PB15/PC8/PC9/PC10/PC11/PC12/PD2` 复用关系要在原理图阶段锁定。 |
| 目标 SWD 引脚 | 不得使用 `PA13/PA14`，这两个脚保留给 CH32 自身调试。 |
| 3.3V 电平策略 | 继续坚持，第一版不要承诺 5V 逻辑兼容。 |
| 下载速度预期 | `300~600KB/s` 成熟目标可以保留，但必须以 CMSIS-DAP v2、批处理、SPI/SDIO 优化为前提。 |

## 14. 一句话结论

`CH32V307RCT6` 是中配无线 DAP 的合理主控选择：硬件资源足够、成本合适、USB HS 集成度高；但项目成败不取决于芯片标称性能，而取决于 USB HS 栈、目标 SWD 引擎、ESP 协处理链路和电源/高速布线是否按工程约束做扎实。
